Configurazione controlli

I controlli sono configurati nella tabella SIAS_CONTROLLI_ANOMALIE.

Campo

Descrizione

CODICE_FUNZIONE

Codice utilizzato dallapplicazione per lidentificazione del controllo

NOME_CONTROLLO

Nome assegnato al controllo

DESCRIZIONE_CONTROLLO

Descrizione di dettaglio (opzionale)

CONFIGURAZIONE_JSON

Parametri di configurazione della funzione di controllo

RETE

Rete a cui è applicato il controllo

ID_TIPO_STAZIONE

Tipo della stazione a cui è applicato il controllo (opzionale)

GRANDEZZA

Grandezza delle misurazioni (vedi configurazione in SIAS_GRANDEZZE_STAZIONE e definizione in SIAS_TIPI_GRANDEZZA)

NOME_UNITA_MISURA

Unità di misura delle misurazioni (vedi configurazione in SIAS_GRANDEZZE_STAZIONE e definizione in SIAS_TIPI_UNITA_MISURA)

ABILITATO

Indica se il controllo deve essere effettuato (1) o no (0)

DATA_ULTIMA_ESECUZIONE

Inserito dallapplicazione dopo lesecuzione del controllo

ESITO_ULTIMA_ESECUZIONE

Inserito dallapplicazione dopo lesecuzione del controllo

 

Controllo Soggiacenza negativa

il controllo verifica se vi sono misurazioni per la grandezza indicata (Soggiacenza) che siano minori del valore indicato in configurazione (es.: 0) nellintervallo di ore indicate in configurazione (es.: 24), allindietro a partire dalla mezzanotte del giorno corrente.

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: SoggiacenzaNegativa

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  "ValoreMinimoSoggiacenza": 0,
  "AmpiezzaPeriodoVerifica": 24
}

Controllo Trasmissione dati

il controllo verifica se vi sono misurazioni per la grandezza indicata (Soggiacenza) nelle ultime ore indicate in configurazione (es.: 48), allindietro a partire dalla mezzanotte del giorno corrente.

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: TrasmissioneDati

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  "OreDaUltimoDatoDisponibile": 48
}

Controllo Plateau soggiacenza

il controllo verifica se vi sono misurazioni per la grandezza indicata (Soggiacenza) nelle ultime ore indicate in configurazione (es.: 72), allindietro a partire dalla mezzanotte del giorno corrente, che oscillano allinterno della soglia indicata.

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: PlateauSoggiacenza

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  "ValoreOscillazione": 0.05,
  AmpiezzaPeriodoVerifica: 72
}

Controllo Spike istantaneo

il controllo verifica se vi sono misurazioni per la grandezza indicata nelle ultime ore indicate in configurazione (es.: 72), allindietro a partire dalla mezzanotte del giorno corrente, con una variazione maggiore di quella indicata tra due ore consecutive.

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: SpikeIstantaneo

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  "ValoreVariazione": 0.30,
  AmpiezzaPeriodoVerifica: 72
}

Versione con numero di misurazioni

La versione con numero misurazioni considera un dato numero di ultime misurazioni invece del periodo.

CODICE_FUNZIONE: SpikeIstantaneoNum

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  "ValoreVariazione": 0.30,
  NumeroMisurazioniVerifica: 24
}

 

Controllo Outlier soggiacenza

il controllo verifica se vi sono misurazioni per la grandezza indicata nelle ultime ore indicate in configurazione (es.: 24), allindietro a partire dalla mezzanotte del giorno corrente, con valori esterni allintervallo media+/- deviazione standard della finestra di controllo (es.: 96).

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: OutlierSoggiacenza

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  AmpiezzaPeriodoVerifica: 24,
  AmpiezzaFinestraControllo: 96
}

Controllo Escursione pozzo

il controllo verifica se vi sono misurazioni per la grandezza indicata nelle ultime ore indicate in configurazione (es.: 24), allindietro a partire dalla mezzanotte del giorno corrente, con valori esterni allintervallo valore minimo/massimo +/- delta minimo/massimo/2 della finestra di controllo (es.: 72).

Configurazione:

Inserire in SIAS_CONTROLLI_ANOMALIE un record con:

CODICE_FUNZIONE: EscursionePozzo

CONFIGURAZIONE_JSON (eventualmente modificando i valori di esempio):

{
  AmpiezzaPeriodoVerifica: 24,
  AmpiezzaFinestraControllo: 72
}
